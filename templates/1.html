Lokalt netværk ¶

⏳ 10 min

Der er opsat en Wi-Fi router i undervisningslokalet eller Tech & Story Lab med et SIM-kort som er tilsluttet TDCs mobilnetværk. Der er kun 32 til 64 computere som kan forbinde til Wi-Fi routeren og forbindelsen har begrænset båndbredde så lad være med at se video over routeren.

  • SSID: KEA_IOT_ROUTER_1
  • Password: MONSTER1

Jeres computer og Raspberry Pi skal være forbundet til det samme netværk for at kunne tilgå den med en fjernadgang. Her er administratorsiden for KEA_IOT_ROUTER_1 med en oversigt over forbundet enheder.

Bemærk at hver enhed har et hostname, for eksempel beitpi, og en IP adresse. Vi kan anvende hostname og ip adressen til at tilgå vores enhed med en fjernadgang fra vores computer over netværket.

Det er også muligt at forbinde jeres Raspberry Pi til Internetdeling med en telefon eller jeres eget Wi-Fi netværk derhjemme. Bemærk at nyere smartphones kan have sikkerhedsbegrænsninger som blokerer for denne mulighed.

Se afsnittet om fjernadgang og skriv følgende for at tilgå netværkskonfigurationen for Wi-Fi og lave ændringer.

sudo nano /etc/wpa_supplicant/wpa_supplicant.conf

Indsæt navn på Wi-Fi netværk og password.

country=DK
ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1
network={
    ssid="your_wifi_name"
    psk="your_wifi_password"
}

Tryk på CTRL+X og Y for at gemme og afslutte.

Alternativt tilslut en skærm med micro HDMI kablet, samt mus og keyboard, for at tilgå netværkskonfigurationen.

Fjernadgang ¶

⏳ 10 min

Når der er sat strøm til Raspberry Pi og den er forbundet til Wi-Fi netværket kan vi tilgå den fra vores computer via fjernadgang med SSH.

SSH

SSH står for Secure Shell og er en krypteret netværksprotokol som vi anvender til at etablere en sikker forbindelse mellem en klient og en server over et netværk.

På macOS er SSH en del af operativsystemet og du kan åbne en ny Terminal for at etablere en forbindelse til din Raspberry Pi via SSH-protokollen. Erstat "hostname" med dit valgte hostname.

ssh pi@hostname.local

Når du skriver dit password vil der ikke være en stjerne (*) for hvert bogstav af sikkerhedsmæssige årsager og første gang du logger ind skal du sige ja til "fingerprint authentication" for SSH-protokollen.

På Windows kan du installere programmet PuTTY (link) til at etablere en forbindelse til din Raspberry Pi via SSH-protokollen. Når du har installeret PuTTY skal du åbne programmet og indtaste IP-adressen eller "hostname.local" (erstat "hostname" med dit valgte hostname) og vælge SSH som protokol. Derefter kan du logge ind med dit brugernavn og password.

Linux kommandoer ¶

⏳ 10 min

Start med at sikre at operativsystemet og diverse softwarepakker er opdateret. Skriv følgende kommandoer en ad gangen via en SSH-forbindelse i et Terminal-vindue.

Du kan skrive "sudo" før en kommando for at udføre kommandoen med root-adgang, som betyder du har super user privilegier og er en slags administratorrettigheder.

sudo apt-get update
sudo apt-get upgrade
sudo reboot

Vent på at din Raspberry Pi har opdateret og genstartet. Derefter åben din fjernadgang igen.

Prøv at skriv følgende kommandoer en ad gangen i et Terminal-vindue via SSH.

uname -a
cat /etc/os-release
cat /proc/version
whoami
ifconfig
iwlist wlan0 scan

Kommandoerne giver svar på hvilket hostname din Raspberry Pi har, hvilket operativsystem, hvilken kerne operativsystemet er bygget på, hvem brugeren er, netværksinformation, de sidste kommandoer som er skrevet i Terminalen og rydde Terminal-vinduet.

For at navigere i stier til filer på tværs af mapper skal vi anvende følgende kommandoer.

Se stien til den nuværende lokation.

pwd

Se alle filer og mapper i den nuværende lokationer.

ls

Opret en ny mappe efterfulgt af et navn på mappen.

mkdir navn-på-mappen

Skift lokation til den nye mappe.

cd navn-på-mappen
pwd

Opret en ny fil efterfulgt af et navn på filen (husk filtype).

touch script.py

Åben en fil efterfulgt af et navn på filen. Tryk CTRL+X for at komme tilbage til Terminalen.

sudo nano script.py

For at gå en mappe tilbage i stien.

cd ..

For at tilgå konfigurationen for Raspberry Pi OS på din Raspberry Pi kan du skrive følgende.

sudo raspi-config

For at åbne den grafiske brugergrænseflade, hvis du har en skærm tilsluttet med HDMI-kablet, kan du skrive følgende. Det er muligt at låne en skærm til brug på KEA i Tech & Story Lab.

startx

Andre brugbare kommandoer er clear og history, som hver især rydder alt indhold i kommandoprompten og viser alle tidligere skrevne kommandoer.

clear
history

Der findes et utal af Linux kommandoer som vi løbende kommer til at få brug for.

Sense HAT ¶

⏳ 45 min

Sense HAT er en ekstra enhed til Raspberry Pi som kan tilsluttes via GPIO-stikkene. Den har en række sensorer, et 8x8 LED matrix display og et joystick.

Tag strømmen fra og sæt Sense HAT ovenpå GPIO-stikkene på din Raspberry Pi. Når Sense HAT sidder fast på GPIO-stikkene kan du slutte strøm til igen.

For at få Sense HAT til at virke skal vi installere følgende pakker.

sudo apt-get update
sudo apt-get install sense-hat

Vi kan nu teste Sense HAT med Python scripts.

Vi starter med at oprette en ny mappe Projects og så henter vi filer med Git (link) fra GitHub (link).

Git er et udviklerværktøj og kommandoen 'clone' betyder at downloade noget kode fra et opbevaringssted, for eksempel GitHub, til din lokale maskine så du kan arbejde med koden.

mkdir Projects && cd Projects
git clone https://github.com/miwnKEA/sense-hat.git
cd sense-hat
ls
touch demo.py
sudo nano demo.py

Vi skriver følgende kode i demo.py.

from sense_hat import SenseHat
sense = SenseHat()
sense.show_message("Hello World!")

Vi gemmer og lukker filen med CTRL+X efterfulgt af Y og Enter.

Eksekver de forskellige Python filer ved at skrive:

python3 demo.py

Prøv at eksekvere resten af filerne i mappen.

ls
cat heart.py
python3 heart.py
python3 rainbow.py
python3 dice.py
python3 pong.py
python3 sensors.py

Ekstra: LCD Touchscreen HAT ¶

⏳ 60 min
🚧 Bemærk det kan give udfordringer med operativsystemet at følge disse trin
⚠️ Hvis det går galt skal du installere operativsystemet forfra på Raspberry Pi
☣️ Det er med høj risiko, så gør det kun hvis du skal bruge LCD Touchscreen HAT i jeres projekt
📅 Denne guide er sidst blevet gennemgået og afprøvet i foråret 2023

LCD Touchscreen HAT er en ekstra enhed til Raspberry Pi som kan tilsluttes via GPIO-stikkene for at få en integreret LCD-skærm og en touchskærm.

Tag strømmen fra og fjern Sense HAT fra din Raspberry Pi så du kan sætte LCD Touchscreen HAT på og slutte strøm til igen.

For at få LCD Touchscreen HAT til at virke skal vi installere følgende pakker. Bemærk at det kan give udfordringer med operativsystemet at både installere softwarepakker for SenseHAT og LCD Touchscreen HAT på det samme Raspberry Pi OS.

cd Projects
git clone https://github.com/MrYacha/LCD-show.git
cd LCD-show
sudo ./LCD35-show

Når din Raspberry Pi er genstartet skal du åbne en fjernadgang igen. Det anbefales at bruge en SSH-forbindelse når vi arbejder med denne HAT.

Det kan være rart at rotere skærmen så din Raspberry Pi kan stå op på langsiden. Åben /boot/config.txt og find indstillingen for dtoverlay nederst.

sudo nano /boot/config.txt
dtoverlay=waveshare35a:rotate=270

Tryk på CTRL+X og Y for at gemme og afslutte. Lav derefter en genstart for at få skærmen til at være roteret.

sudo reboot

Vi kan kalibrere touchskærmen så den passer til rotationen.

sudo apt-get install xinput-calibrator
DISPLAY=:0.0 xinput_calibrator

Gem output fra kalibreringen som ligner det her.

Section "InputClass"
Identifier	"calibration"
MatchProduct	"ADS7846 Touchscreen"
Option	"Calibration"	"197 3904 3939 225"
Option	"SwapAxes"	"1"
EndSection

Indsæt output fra kalibreringen i denne fil og åben filen med sudo nano.

sudo nano /usr/share/X11/xorg.conf.d/99-calibration.conf

Tryk på CTRL+X og Y for at gemme og afslutte. Lav derefter en genstart for at få kalibreringen til at virke.

sudo reboot

Vi skal installere forskellige moduler for at få Raspberry Pi til at starte i "kiosk"-mode og vise en hjemmeside på LCD Touchscreen HAT. Kiosk betyder at LCD Touchscreen HAT kun viser det indhold på skærmen som vi har indstillet den til at gøre og brugeren kan ikke klikke hen på andre sider, men brugeren kan godt navigere rundt på hjemmesiden.

sudo apt-get install --no-install-recommends xserver-xorg
sudo apt-get install --no-install-recommends xinit
sudo apt-get install --no-install-recommends x11-xserver-utils

sudo apt-get install chromium-browser
sudo apt-get install matchbox-window-manager xautomation unclutter

Lav en fil som hedder kiosk i mappen kiosk i mappen Projects.

cd ~/Projects/kiosk && touch kiosk

Åbn filen kiosk i nano.

sudo nano kiosk

Indsæt koden nedenfor ind i filen.

#!/bin/sh

# disable DPMS (Energy Star) features.
xset -dpms

# disable screen saver
xset s off
xset s noblank

# don't blank the video device
matchbox-window-manager -use_titlebar no &

# hide X mouse cursor unless mouse activated
unclutter &
chromium-browser --display=:0 --kiosk --incognito --window-position=0,0 https://kea.dk

Tryk på CTRL+X og Y for at gemme og afslutte.

Ændr rettighederne på filen kiosk, så den kan køres.

sudo chmod 755 kiosk

Åben filen .bashrc i nano og kopier koden nedenfor ind i filen.

sudo nano ~/.bashrc
xinit /home/pi/Projects/kiosk/kiosk -- vt$(fgconsole)

Tryk på CTRL+X og Y for at gemme og afslutte.

Åben Raspberry Pi konfigurationen og lav følgende ændringer.

sudo raspi-config

# vælg Display Options, D2 Underscan, Enable compensation? [No]

# vælg System Options, S5 Boot / Auto Login, [B2 Console Autologin]

# afslut og genstart

sudo reboot

LCD Touchscreen HAT vise hjemmesiden www.kea.dk når din Raspberry Pi er genstartet. URL-adressen kan ændres til en Flask applikation på det lokale netværk, for eksempel http://mikkelpi.local:5000.
